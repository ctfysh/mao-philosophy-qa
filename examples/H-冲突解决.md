# 冲突解决——跟同事吵了一架，怎么收场？

> 阿强和老李在项目评审会上吵起来了。阿强的技术方案被老李当众批得一文不值，阿强没忍住回怼了"你懂什么"。会后两人不说话，项目卡住了。

```
用户：我话一说出口就后悔了。但老李在会上那个态度也很过分——完全否定了我两周的工作。现在项目卡住了，他的模块跟我的有依赖，他不理我我推进不了。
```

**毛选原理**：区分对抗性矛盾和非对抗性矛盾（《关于正确处理人民内部矛盾的问题》L14568）。

**分析**：你先判断一下——你和老李的矛盾是"他对你有个人敌意"（对抗性），还是"做事方法不同、经验体系不同"（非对抗性）？你说老李做了这套系统很多年，觉得只有他的方式才是对的。听起来更像是经验主义导致的固执，而不是针对你这个人。这意味着这个矛盾不需要"赢"来解决，而是需要"找到一个方案让两人都能接受"。

**行动指引**：你的目标是什么？是"让他认错"，还是"项目能推进，以后不尴尬"？

```
┃ ▼ 继续追问 ┃
如果只是为了项目能推进，你愿意主动去找他吗？还是觉得必须先等他来道歉？
```

> 阿强确认了目标：项目要推进，关系要修复。

```
用户：我想要项目推进下去，也不想以后每天见面尴尬。但要我完全认错也不甘心。
```

**毛选原理**：团结—批评—团结（《关于正确处理人民内部矛盾的问题》L14585）。从团结的愿望出发，经过批评使矛盾得到解决，从而在新的基础上达到新的团结。

**分析**：第一步不需要认错，只需要传递一个信号——"我不是把你当敌人"。你可以主动找他，不是去道歉，而是说："老李，上次会上我话重了，但项目还得往前走，我想听听你的想法。"

注意你的目的不是说服他接受你的方案，而是**了解他否定你的依据**。带着三个问题去：你觉得我方案最大的问题在哪？如果是你会怎么做？我们综合一下有没有可能？

**行动指引**：明天主动找他，问这三个问题。把"谁对谁错"的框架换成"哪个方案更适合当前条件"的框架。

```
┃ ▼ 继续追问 ┃
你问得出这三个问题吗？还是觉得问了就像认输了？
```

> 阿强第二天找老李谈了，发现两人的方案其实可以结合——用老李的架构做底层，用他的设计做前端。

```
用户：我去找他了。问了那三个问题，他态度软下来了。最后发现两个方案其实可以结合——用他的架构做底层，用我的前端。结果比我们各自的方案都好。
```

**毛选原理**：矛盾推动发展——经过斗争（良性批评）达到了新的认识水平。

**分析**：你自己体验了"团结—批评—团结"的完整循环。通过这次冲突，你们看到了各自方案之外的第三种可能。如果当时没人主动迈出第一步，这个更好的方案永远不会出现。

**行动指引**：以后遇到类似的分歧，先把"谁对谁错"的想法停下来，问自己一句：我到底想要什么？是要赢，还是要一个更好的结果？

```
┃ ✓ 追问总结 ┃
1. 核心矛盾：把"谁对谁错"的意气之争，转为"什么方案更适合"的事实讨论
2. 行动方向：主动不代表认输，而是推动事情往想要的方向走
3. 以后可先问自己：我是想要赢，还是想要把事情做好？
```
