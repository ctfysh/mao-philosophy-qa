# 冲突解决——跟同事吵了一架，怎么收场？

> 阿强和老李在项目评审会上当众吵起来了。阿强做了两周的技术方案被老李批得一文不值，阿强没忍住回怼了"你懂什么"。会后两人不说话，阿强的模块和老李的模块有依赖关系，项目卡住了。阿强又后悔又憋屈。

```
用户：我做完方案熬了两夜，会上本来以为能过，结果老李一上来就说"你这方案根本不靠谱，重做"。当着项目总监和三个同事的面。我当时脑子一热就说"你懂什么，这技术你都没用过"。话一出口就后悔了，但收不回来了。会后他直接走了，我去茶水间他看见我就转身。现在项目卡在我等他接口，他不理我。
```

**毛选原理**：区分对抗性矛盾和非对抗性矛盾（《关于正确处理人民内部矛盾的问题》L14568）。人民内部的矛盾不是敌我矛盾，解决方法不是"消灭对方"而是"讨论协商"。

**分析**：你和老李的矛盾属于哪一类？你和他有共同目标——都是让项目做成。分歧在"用什么方式做"。这是典型的非对抗性矛盾。但你们当前的处理方式（互不搭理）正在把它推向对抗化——项目一停，共同利益就开始受损了。非对抗性矛盾的解法不是"谁认输"，而是找到双方都能接受的新方案。

**行动指引**：今晚别在微信上发消息。情绪还在的时候说话容易被对方读出攻击性。

```
┃
┃ ▼ 继续追问
┃
你说老李"这技术你都没用过"——他具体是怎么批你方案的？是直接否定，还是指出了什么问题你没接住？
```

```
用户：他大概说了三条。第一说我选的新框架团队没人熟，学习成本高。第二说现有系统的接口协议跟这个框架不兼容，改造成本太大。第三说项目两个月后要交付，折腾不起。我当时觉得他根本没认真看我的设计——我方案里写了兼容层，他好像翻都没翻到那页。
```

**毛选原理**：矛盾的特殊性——具体问题具体分析（《矛盾论》L3200）。每个矛盾都有它独特的一面，照搬一般结论不能解决具体问题。

**分析**：你刚才说的很有意思。老李提了三条具体理由，你也知道其中一条（兼容层）他已经漏看了。这意味着什么？意味着你们的争论在信息不对称的基础上进行——他没看完你的方案，你没听进去他的顾虑，两个人在各自的信息茧房里吵架。你说他"根本不懂"，他可能确实没看完你的方案；但你说他"故意针对你"，这个判断还需要更多证据。

**行动指引**：先区分一下——老李没看完方案是"工作疏忽"还是"故意刁难"？这两个性质完全不同。

```
┃
┃ ▼ 继续追问
┃
老李来公司多少年了？他跟你之前的关系怎么样——在这件事之前，你们有过正常的工作交流吗？
```

```
用户：他来公司八年了，我才两年。之前我们关系还行，偶尔一起午饭。他还教过我几次系统的事。就是因为以前对他印象不错，会上被他一顿批才特别受不了——我觉得"你以前对我挺好，怎么会上这样"。
```

**毛选原理**：现象与本质——不要被一时的现象迷惑，要透过现象看本质（《在中国共产党全国代表会议上的讲话》L13044）。

**分析**：你再想一想：老李在会上当众否定你的方案，会不会恰恰因为他觉得自己有责任把关？他做了八年，对这个系统的坑太熟了。你说"他连兼容层都没翻到"——他可能确实翻漏了，这让你觉得他"不认真"或"针对你"。但一个做了八年的人，有必要在工作群殴会议上故意整一个后辈吗？他有这个动机吗？

**行动指引**：把"他对我不满"这个推论放在一边，先检查另一个可能性——他是不是真的只是着急，怕项目出问题，表达方式太硬了？

```
┃
┃ ▼ 继续追问
┃
就算他不是针对你个人——他在会上那个态度，你感受上最受不了的是什么？是"他说得不对"，还是"他那个居高临下的语气"？
```

```
用户：都有。但最受不了的是他当着总监和同事的面，直接说"你这方案根本不靠谱，重做"。没有一句"我觉得"或者"你看这里是不是有问题"，一上来就否定了。我觉得他不尊重我两周的付出。
```

**毛选原理**：言者无罪，闻者足戒（《整顿党的作风》L8802）。听到不同的意见，先不要急于反驳，冷静下来看有没有合理的成分。

**分析**：你说到关键点了——"不尊重"才是你真正生气的原因，而不是他提了三条技术理由。技术理由是议题，不尊重是情绪。但有意思的是：如果他是冲着解决项目问题来的（只是方式太硬），那他的三条理由里有没有值得听的？你自己也承认第一条（学习成本）和第二条（接口兼容）是真实存在的风险，第三条（交付时间）也是事实。他的方式伤害了你，但内容不一定全错。

**行动指引**：把"他这个人不好"和"他说的话有没有道理"分开。你可以不接受他的态度，但不代表他的每句话都该被否定。

```
┃
┃ ▼ 继续追问
┃
如果抛开语气不谈，他提的三条风险你觉得有哪条是需要认真对待的？
```

```
用户：学习成本和接口兼容两条确实存在。我在方案里写了兼容层，但确实会增加开发时间。时间上两个月确实紧。只是我觉得这些问题是可解决的，不是"方案不行"。
```

**毛选原理**：两分法/两点论（《加强相互学习，克服固步自封、骄傲自满》L18940）。任何事物都有两面，正面看三条，反面看三条。

**分析**：你现在已经开始客观了——你承认风险存在，同时也看到解决方案。这才是讨论问题的态度。但注意，你在会上没机会把这些说清楚，因为你们两个在情绪层就卡住了。他一上来就否定，你一听否定就炸了。如果他说"这个方案有三个风险我们来讨论一下"，你大概率会心平气和地解释兼容层的设计。

**行动指引**：下次有人否定你，先问自己一句：他说的是"方案不行"还是"方案有风险"？前者是结论，后者是讨论邀请。

```
┃
┃ ▼ 继续追问
┃
回到那天会上——你说总监也在场。你觉得老李上来就否定你的方案，这跟他和总监之间的关系有没有关系？
```

```
用户：你这么一说……有可能。总监在的时候他说话本来就比平时大声。而且他最后补了一句"我在这系统上栽了三年跟头我清楚"。这句话好像不是说给我听的，是说给总监听的——意思是"你别信新人的，得信我"。这么一想他可能不是针对我，是怕总监觉得他不够负责。
```

**毛选原理**：主要矛盾与矛盾的主要方面（《矛盾论》L3252）。复杂事物中有一对矛盾起主导作用，抓住它就抓住了全局。

**分析**：好的，你现在离真相近了一大步。你和老李的矛盾是表象。真正的主要矛盾是什么？是项目两个月后要交付，但目前技术和时间之间有不匹配。老李怕冒进风险，你怕老方案低效。你们俩的目标其实一样：项目成功。分歧在路径选择。老李在会上"过分"的表现，本质是他保护自己安全感的方式——人在感觉失控时就会声音变大。

**行动指引**：你现在的对手不是老李。你们共同的"对手"是两个月后的交付压力。问题是：你愿不愿意把他从"对手"变回"队友"？

```
┃
┃ ▼ 继续追问
┃
如果现在有一个方案既能用到你的设计思路，又能让他觉得核心技术路径在他的掌控范围内，你愿意让步多少？
```

```
用户：我其实不是非要全部用新框架。让我完全放弃也不甘心，毕竟熬了两夜。但如果能用一部分，比如只在展示层用新方案，底层还是用他熟悉的架构——我可以接受。问题是现在谁也不找谁，连谈的机会都没有。
```

**毛选原理**：团结—批评—团结（《关于正确处理人民内部矛盾的问题》L14586）。从团结的愿望出发，经过批评或争论，达到新的团结。起点决定终点。

**分析**：你现在卡在"谁先迈第一步"上。你觉得应该是他来找你（因为他批得太狠），他觉得应该是你来找他（因为你回了"你懂什么"）。两个人都等着对方先动——项目就这么耗着。注意一个关键点：你们俩都认为"先找对方=认输"。这个等式本身就是错的。主动沟通不是认输，是主动创造解决问题的条件。你去找他不是去道歉，是去"把项目重新推起来"——这是你的项目，你的责任，你也有一部分方案在里面。

**行动指引**：第一步不要求自己"大度道歉"。只需要一个最小的接触动作，传递一个信号。

```
┃
┃ ▼ 继续追问
┃
明天你能做的最小一号行动是什么——不是"和好"，只是"重新建立连接"？
```

```
用户：最小的话……我可以拿着我的方案，圈出展示层那部分，走过去问他："这部分我想用新框架做底层，你看接口那边能不能配合？"——不提上次的事，只谈技术。但我怕他还在生气，直接给我冷脸。
```

**毛选原理**：内因外因关系（《矛盾论》L3130）。内因是变化的根据，外因是变化的条件——你能控制的只有你自己。

**分析**：你怕他给冷脸——这个"怕"是你的内因。他给不给冷脸是他的反应（外因），你控制不了。但你能控制的是：第一，你确实去找他了；第二，你说的话是工作内容，不是情绪发泄；第三，你的姿态是"我想推进项目"。如果他真的继续冷脸，那不是你的错——你做了你能做的部分。而且你想一想：你一个晚辈主动去找他谈技术，他在总监和同事眼里反而显得小气。

**行动指引**：明天上午拿方案去找他，用技术问题开场："老李，底层接口配合的事想跟你碰一下。"如果他一开口就是"上次你说我不懂"——不要接这个话头，直接说"上次我话说重了，我们先把项目往前推一下行吗？"

```
┃
┃ ▼ 继续追问
┃
你说"怕他冷脸"——你最怕的是他冷脸本身，还是怕自己再一次控制不住情绪？
```

```
用户：怕自己控制不住。我上次就是没忍住才吵起来的。万一他再甩一句难听的，我可能又炸了。我不想再后悔一次。
```

**毛选原理**：放下包袱，开动机器（《学习和时局》L7776）。先卸下精神负担——怕丢脸、怕再次失控、怕被看低——思考才能活起来。

**分析**：你身上的包袱不轻。一层是"怕他拒绝"，二层是"怕自己再失控"，三层是"如果这次搞砸了就证明我情商不行"。三层包袱叠在一起，你当然迈不动腿。但你反过来想——你现在"什么都不做"，项目一天天卡着，这本身就是在积累更大的后果。总监不会一直等着你们自己消化情绪。你主动去谈，就算谈崩了，至少你在总监那儿的印象是"阿强想解决问题"。你不去，印象就是"两个人在拖项目"。

**行动指引**：今晚给自己做一个小练习：写下最坏的情况（他狠狠拒绝你），和最好的情况（他态度软下来一起讨论方案）。最坏的情况你能不能承受？能的话就去。

```
┃
┃ ▼ 继续追问
┃
如果最坏的情况发生了——他拒绝沟通——你打算怎么应对，让事态不至于更糟？
```

```
用户：如果他说"没什么好谈的"，我就说"那我把我的部分先放一放，等项目总结会上再讨论"。先撤一步，不硬顶。然后去找项目总监，说明我在尝试沟通但对方不配合，请求总监协调。至少我做了该做的，责任不在我这边了。
```

**毛选原理**：不打无准备之仗（《关于重庆谈判》L9430）。预判各种结果并做好预案，行动才有底气。

**分析**：好，你现在有了预案。最坏的情况你已经想好怎么应对了——"先撤一步，向上升级"。这意味着你明天去敲门已经没有心理负担了。因为你知道：最低线是项目能推进（即便需要总监介入），中线是老李态度缓和你们私下解决，上线是两人合作出一个结合方案。三条线你都做了准备。

**行动指引**：明早去公司，趁他还没被邮件和会议淹没的时候去找他。开口记得强调"项目推进"，不要提"谁对谁错"。

```
用户：九点半吧，他到公司一般先泡咖啡。我在茶水间"偶遇"他，自然一点。第一句话就说："老李，上次会上你说兼容性问题我回去想了想，确实有道理。我在展示层用新框架，底层接口还是走你那套，你看接口上能配合吗？"
```

**毛选原理**：把问题摆到桌面上来（《党委会的工作方法》L11002）。有话当面讲，有问题公开谈，不在背后猜测。

**分析**：你这个开场很好。有三个关键动作：第一，你用"你说得确实有道理"给了他台阶——不需要认错，只需要承认对方的一部分观点有价值；第二，你把问题从"要不要用新框架"缩小到了"接口能不能配合"，范围变窄了，对方容易答应；第三，你用的场合是"偶遇"，不是正式会谈，压力小。这叫"原则性和灵活性相统一"——你的原则是"展示层用新方案不想放弃"，灵活的是"沟通方式和让步空间"。

**行动指引**：记住，你这次谈话的目标不是"让他喜欢你"，而是"项目能往前走一步"。

**分析**：你看，你准备的"展示层方案"和"承认他说得有道理"就是转化条件。有意思的是，他也在做他的工作——他回去查了框架资料。这说明什么？说明他对你的方案并不是完全否定，只是当众被后辈怼了，面子下不来。你给了台阶，他就顺着下来了。你们俩的矛盾从"要不要用新框架"转化成了"底层接口怎么改"，从一个原则之争变成了一个技术问题。技术问题好解决，原则问题才难解决。

**行动指引**：记住今天的感觉——主动带来的不是"丢面子"，而是局面开始动了。

**毛选原理**：矛盾的同一性与斗争性——矛盾双方在一定条件下的互相转化（《矛盾论》L3292）。冲突可以转变为合作，关键在于转化条件的创造。

**分析**：注意几个细节。第一，你主动让步之后他也主动了——他回去查了框架资料，说明他在你们冷战期间也在想你的方案。第二，"拍肩膀"这个动作很重要——这是身体语言在说"没事了"。第三，他说"回头把你方案再发我一份"——这是他在重新打开你的方案，愿意重新评估。三个月前你们之间是"你的方案不行"，现在是"我再看看你的方案"——这个转变不是因为谁赢了，而是因为你创造了让他能下台阶的条件。

**行动指引**：以后遇到类似冲突，可以先用"你说得有道理"开头——承认对方的一部分观点，不表示你全盘认输，只是让对话从零和变成合作。
