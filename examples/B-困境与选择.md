# 困境与选择——复杂局面无从下手？

> 阿杰创业两年，团队 8 人。最近同时爆发三个问题：核心工程师要离职、投资人对赌协议快到期、产品用户留存率不到 10%。他每天被这三件事来回拉扯，什么都推进不了。

```
用户：我现在每天一睁眼就有三件大事，每件都很紧急，顾了这个顾不上那个，感觉随时要崩盘。
```

**毛选原理**：主要矛盾（《矛盾论》L3252）。"在复杂事物的发展过程中，有许多矛盾存在，其中必有一种是主要的矛盾，由于它的存在和发展规定或影响着其他矛盾的存在和发展。"

**分析**：你把三件事当作并列的危机同时应对，但如果仔细看，它们之间有一条因果链——留存低 → 买量成本高 → 烧钱快 → 投资人心态紧张 → 工程师看不到希望要离职。三件事其实是一根链条上的不同环节，而**留存是源头**。

**行动指引**：先不急着解决任何一件事。先调查——找到 5 个注册后流失的用户，问问他们为什么不用了。这个动作不需要技术配合，不需要花钱，你一个人明天就能做。

```
┃ ▼ 继续追问 ┃
工程师离职和投资人紧张，这两个问题哪个是先出现的？
```

> 阿杰仔细回想后意识到，工程师是在看到留存数据连续三个月下滑后才开始找机会的。

```
用户：我回想了一下，工程师是看了上季度的留存数据后才开始找工作的。投资人那边也是因为留存上不去、买量成本越来越高才催我们。
```

**毛选原理**：调查研究（《反对本本主义》L1685）。"调查就像'十月怀胎'，解决问题就像'一朝分娩'。调查就是解决问题。"

**分析**：你现在看到的"三个危机"，本质是一个问题（留存低）引发的连锁反应。如果你解决了留存，另外两个会自然松动。你之前觉得"每件都很紧急"是因为你把症状当成了病根。现在你知道了病根在哪，接下来不是分析它，而是去了解它——你了解你的用户为什么走吗？

**行动指引**：明天列一份最近一个月注册后三天内没再登录的用户名单，挑 10 个出来。逐个联系，就问三个问题：当初为什么注册？使用中遇到什么困难？后来为什么没再来？

```
┃ ▼ 继续追问 ┃
你打算用发消息还是打电话的方式联系他们？
```

> 阿杰联系了 7 个用户，发现两个他完全没想到的问题：注册流程太复杂（要填 6 个字段），核心功能藏得太深用户找不到。

```
用户：我跟 7 个流失用户聊了，发现注册流程太复杂了，要填好多字段。还有我们最核心的功能，用户进来根本找不到在哪。这两个改起来不需要多少开发量。
```

**毛选原理**：战略上藐视、战术上重视（《关于帝国主义和一切反动派是不是真老虎的问题》L16272）。整体上敢于胜利，具体上严谨对待。

**分析**：你之前觉得"留不住用户"是天大的难题，调查完发现只是两个具体问题：注册门槛和功能可见性。这就是为什么调查比空想有用——它把抽象的恐惧变成了具体的待办事项。

**行动指引**：第一优先级简化注册流程，第二优先级加新手引导指向核心功能。两周内上线。

```
┃ ✓ 追问总结 ┃
1. 核心矛盾：把"三个并列危机"重新理解为"一个根本问题+两个连锁反应"
2. 行动方向：调查先于决策，找到主要矛盾再集中解决
3. 以后可先问自己：这些问题是并列的，还是一根链条上的不同环节？
```
