# 困境与选择——复杂局面无从下手？

> 阿杰创业两年，8 人 SaaS 团队，做中小企业数据分析工具。最近一个月三件事同时炸开：核心工程师老周提了离职、投资人对赌协议还有三个月到期、产品月留存掉到不到 10%。他被三股力量往三个方向拽，什么事都推不动。这天晚上十一点，他坐在办公室，打开了对话窗口。

```
用户：我现在每天一睁眼就有三件大事要处理。核心工程师要走，投资人对赌快到期，产品留存不到 10%。每件都火烧眉毛，我顾了这个顾不上那个。昨天开了四个会，一个结论都没有。凌晨三点还在想怎么同时解决这三件事，越想越睡不着。
```

**毛选原理**：主要矛盾（《矛盾论》L3252）。"在复杂事物的发展过程中，有许多矛盾存在，其中必有一种是主要的矛盾，由于它的存在和发展规定或影响着其他矛盾的存在和发展。"

**分析**：你现在觉得"三件事都很紧急，必须同时解决"。但直觉不一定是准的——如果这三件事不是并列的，而是一根链条上的不同环节呢？投资人紧张和工程师要走，可能不是独立的问题，而是同一个病根的并发症。你急着同时灭火，但有没有想过，三个火苗其实是从同一个火源溅出来的？

**行动指引**：今晚不做任何决定。只做一件事——排出这三件事出现的时间顺序。哪件最先冒头，哪件随后出现，哪件是最近才爆发的？

┃
┃ ▼ 继续追问
┃
你刚才说"留存不到 10%"——这个数字是从什么时候开始往下掉的？

---

```
用户：我排了一下——留存三个月前从 20% 开始掉，到现在不到 10%。大概掉了一个月之后，投资人开始在月会上频繁问数据，话里话外提对赌条款。老周是一个半月前开始状态不对的，前几天正式提了离职。
```

**毛选原理**：现象与本质（《实践论》L2861）。"感觉到了的东西我们不能立刻理解它，只有理解了的东西才能更深刻地感觉它。"

**分析**：时间线已经把结构说清楚了：留存出问题 → 投资人紧张 → 核心成员失去信心。三件事不是平等的三个敌人，是同一个病根引起的三级症状。你觉得自己面对的是"三个问题"，其实只有一个问题——产品留不住人。投资人的紧张和工程师的离职，都是这个病挂的急诊号。

**行动指引**：明天上午把过去三个月的留存曲线调出来，标出每一次版本发布的时间点。看看下降是匀速的，还是跟某个版本发布强相关的。这个动作花不了 20 分钟，但能告诉你一些你一直在忽略的东西。

┃
┃ ▼ 继续追问
┃
你之前为改善留存做过哪些具体动作？

---

```
用户：我做过挺多的——加了用户积分系统、上了两个版本的功能迭代、搞了一次老用户召回邮件。每件事都花了不少精力，但数据就是不动。说实话我现在对"保留存"这件事有点绝望了。总觉得该做的都做了，没用。
```

**毛选原理**：内因与外因（《矛盾论》L3584）。"事物发展的根本原因，不是在事物的外部而是在事物的内部，在于事物内部的矛盾性。"

**分析**：你说的"该做的都做了"——可我仔细看了你做的这些事：积分系统是外部激励，功能迭代是你猜的需求，召回邮件是事后补救。全都是你在办公室里想出来的，全都绕着问题的外围打转。你做了很多动作，但没有一个动作是去问用户"你为什么不来了"。

这就是内因和外因的区别——你一直在外部打转：发邮件、加功能、搞积分。但用户流失的内因是你产品本身有毛病。你不找到这个内因，外面敲敲打打再多也没用。

**行动指引**：明天不做任何产品改动。挑 10 个最近一个月注册后流失的用户，逐个发消息或打电话，问清楚三件事：为什么注册？遇到了什么困难？为什么后来不来了？

┃
┃ ▼ 继续追问
┃
你心里其实已经猜到用户流失可能是什么原因了，对吧？但你一直没有去验证。为什么？

---

```
用户：你说到我痛处了。我确实猜过——我猜注册流程太复杂，也猜过功能不好找。但为什么没去验证呢？可能是因为……怕确认了这些问题之后，就得花更多时间去改，而我现在最缺的就是时间。还不如先做点"看上去在推进"的事，至少能让自己觉得今天没白过。
```

**毛选原理**：逃避矛盾（反面教训，《矛盾论》L3415）。"矛盾是普遍的、绝对的，存在于事物发展的一切过程中，又贯串于一切过程的始终。"

**分析**：你这段话非常值得停下来好好看。你不是不知道问题在哪，你是不敢面对——因为你怕验证了问题之后，就得去解决它，而你觉得自己没有资源去解决。所以你用"做一些看起来在推进的事"来麻痹自己。积分系统、功能迭代、召回邮件——它们的作用不是改善留存，是让你自己感觉好一点。

但矛盾不会因为你回避它就消失。它只会累积，到爆发的时候更猛烈。你花在积分系统上的每一分钟，都是本应用来搞清楚用户为什么离开的时间。

**行动指引**：今晚就做一件事——别想"我要不要做用户访谈"，直接拿起手机，给一个流失用户发微信。突破心理障碍的最好办法不是想清楚再做，是做着做着就想清楚了。

┃
┃ ▼ 继续追问
┃
如果用户说的原因是你改不了的——比如价格太贵了、功能太弱了——你打算怎么办？

---

```
用户：你让我今晚就约用户聊，但我明天早上还有投资人会议要准备，下午要跟老周 1on1，晚上还要改 BP。你说的道理我都懂，但你能不能别安排在今天？我真的排不开了。
```

**毛选原理**：主要矛盾与次要矛盾的关系（《矛盾论》L3270）。"用不同的方法去解决不同的矛盾，这是马克思列宁主义者必须严格地遵守的一个原则。"

**分析**：你现在是在用"忙"来当盾牌。但你仔细想想——你明天要做的三件事（投资人会议、老周 1on1、改 BP），每一件本质上都是在应对同一个问题：产品没有起色。你不去搞清楚用户为什么走，明天的投资人会议上你说什么？跟老周 1on1 你谈什么？BP 改了哪一页能让你睡好觉？

**忙和有效率是两回事**。你现在忙得团团转，但没有一件事是往病根上动刀子的。我不是让你不准备投资人会议——我是让你在准备会议之前，先花 30 分钟拿到一个真实的信息。哪怕今晚只联系一个用户，明天你跟投资人说话的时候，底气都不一样。

**行动指引**：现在就定个闹钟——30 分钟。打开你们产品的后台，找到最近一周注册但没活跃的用户，选一个最像你目标客户的，发条微信或打个电话。30 分钟后不管结果如何，你再去准备 BP。

┃
┃ ▼ 继续追问
┃
你刚才说"怕验证了问题就得花时间改"——如果验证出来问题确实很难改，你打算怎么办？

---

> 阿杰盯着那段话看了很久。然后他翻出后台，找到了一个流失用户，发了条微信语音。对方没接。他又换了一个，这次接了。聊了十来分钟。挂了电话之后他在对话框里打了又删、删了又打，最后发了条消息过来。

```
用户：我刚才跟一个流失用户打了电话。他说他注册的时候填了六七个字段，烦了，关了。就这么简单。不是因为价格、不是因为功能不够——就是注册太烦了。六七个字段啊，我回想了一下，好像真的有这么多。这个问题我三个月前就应该发现，但从来没去问过。
```

**毛选原理**：实践出真知（《实践论》L2895）。"无论何人要认识什么事物，除了同那个事物接触，即生活于（实践于）那个事物的环境中，是没有法子解决的。"

**分析**：注意你现在的状态变化——十分钟前你说"我没时间"，现在你不仅有了时间，还拿到了一个你花了三个月都没发现的信息。你从"猜测"进入了"知道"。哪怕只有一个用户，你的决策基础已经不一样了。

你还说了"这个问题三个月前就该发现"——对，但你当时没发现，是因为你一直在想，没有去问。这不是能力问题，是方法问题。现在你知道了方法，以后就不会再犯同样的错。

**行动指引**：继续联系。今晚再打两个电话，凑够三个。不需要等全部联系完再行动——如果第一个用户说的注册问题和第二个用户说的功能入口问题对得上，你就已经找到方向了。

┃
┃ ▼ 继续追问
┃
如果接下来的两个用户说的原因完全不一样——一个说价格太贵，一个说功能太少——你怎么办？你相信哪个？

---

> 阿杰又打了两个电话。三个用户说的原因不完全一样，但有两个提到了同一个问题：注册流程太长，没耐心填完。另一个说的是价格，但对阿杰来说，两个对得上的信号已经够了。

```
用户：我又打了两个。三个里面有两个说了注册的问题，另一个说价格贵但願意等免费试用结束再决定。我觉得注册这个问题确实坐实了。还有一个人提到进来以后不知道点哪里，可能功能入口也有问题。我现在有点激动又有点不是滋味——这么简单的问题，我为什么花了三个月？
```

**毛选原理**：调查研究（《反对本本主义》L1685）。"调查就像'十月怀胎'，解决问题就像'一朝分娩'。调查就是解决问题。"

**分析**：你花了三个月猜，猜来猜去都是错的。你跟用户打了三个电话，就发现了两个真正的问题——而且这两个问题改起来不比你搞积分系统花的时间多。这就是为什么"调查就是解决问题"。你之前觉得调查浪费时间，现在看到了吧——不调查才是最大的时间浪费。

还有一个变化你注意到了吗？你之前对留存是"绝望"的，因为你闭着眼睛乱打，打不中就越来越挫败。现在你知道了问题在哪，虽然还没解决，但心态已经从"没救了"变成了"有方向"。这就是调查带来的力量。它不是直接给你答案，但它让你从恐惧走到了行动。

**行动指引**：剩下的 7 个用户继续联系，把样本扩到 10 个。同时开始整理两个发现的改动方案——注册流程简化和新手引导——把方案和时间预估写清楚。

┃
┃ ▼ 继续追问
┃
这两个问题，你自己注册过产品吗？如果你自己都没走过一遍完整的注册流程，你怎么能发现它有问题？

---

```
用户：你说的我哑口无言。我其实从来没完整走过一遍自己产品的注册流程——因为我是创始人，账号是技术直接开的。新手引导也是，我一直觉得我们的功能设计得很直观，没想到真实用户进来根本看不懂。我天天在用数据分析面板，但一个新用户打开这个页面是完全空白的，他连第一步该做什么都不知道。
```

**毛选原理**：实践论（《实践论》L2910）。"你要有知识，你就得参加变革现实的实践。你要知道梨子的滋味，你就得变革梨子，亲口吃一吃。"

**分析**：这是一个典型的创始人盲区。你太熟悉你的产品了，以至于你完全忘了新手是什么感受。你天天在里面看数据、操作各个模块，当然觉得功能很好找。但一个第一次打开的用户，面对一个空白界面，连这个工具是干什么的都说不清楚。

而且你再想想——你每天都在替用户做决策，都在想他们需要什么。但你从来没把自己放在用户的位置上。如果你连自己的产品都没完整注册过一遍，你怎么可能做出用户真正需要的东西？这不是一个产品问题，这是一个方法论问题。

**行动指引**：明天拿一台没装过你们产品的手机，从应用商店搜索开始，完整走一遍新用户注册到使用核心功能的流程。录屏。然后你自己看一遍回放，数数一个全新用户从打开到完成核心任务需要点几次按钮、填几个框、花几分钟。

┃
┃ ▼ 继续追问
┃
你发现没有——你跟用户聊了三个电话，就知道了你在办公室坐三个月都不知道的事。这件事本身告诉了你什么？

---

> 阿杰花了三天时间，联系了所有 10 个流失用户，又用自己的手机把注册流程完整录了一遍。他还让一个不懂产品的朋友当着他的面试了一次，在那位朋友点了五六下还没找到核心功能的时候，他坐在旁边一句话都说不出来。

```
用户：我把注册字段从 6 个砍到了 2 个，加了一个弹窗新手引导指向核心功能。两个改动花了一周时间上线了。留存从 8% 涨到了 11%——虽然离目标还远，但这是三个月来第一次止跌回升。但是……老周那边一直没进展。他要配合做一个后端接口的调整，我催了几次他都说"在做"，进度很慢。我又卡住了。
```

**毛选原理**：矛盾转化（《矛盾论》L3632）。"矛盾的主要和非主要的方面互相转化着，事物的性质也就随着变化。"

**分析**：你解决了第一个矛盾，新的矛盾就冒出来了。之前的主要矛盾是"不知道用户为什么走"，现在变成了"知道怎么改但核心成员配合不上"。这是好事——矛盾转化说明你在前进，说明你从认知阶段进入了执行阶段。

但执行阶段的矛盾性质不一样。老周配合不上，不是因为技术难度大，而是因为信任还没恢复。你之前三个月的"瞎忙"让他对公司失去了信心，你现在一周的改善不会自动换回他的信任。信任的恢复需要时间，也需要你有意识地去经营。

**行动指引**：不要催老周。先把需要他配合的模块拆成最小的独立单元，写出明确的 PRD 和技术评估，一次性给他。告诉他时间他自己安排，你只想知道他需要多久。给他掌控感，而不是压迫感。

┃
┃ ▼ 继续追问
┃
老周看到留存回升的数据了吗？你有没有主动跟他同步过进展？

---

```
用户：我跟老周又谈了一次。我把需求文档给他了，他说他可以做，但得先把手上另一个紧急 bug 修完，至少需要一周。我不确定他是不是在拖，但又没法说他不忙——他确实每天在写代码。我把留存数据调出来给他看了。他看了一眼说"嗯有起色"，然后继续低头看电脑。我觉得他还在观望。
```

**毛选原理**：内因与外因的转化（《矛盾论》L3601）。"外因是变化的条件，内因是变化的根据，外因通过内因而起作用。"

**分析**：你是对的，他确实在观望。你的数据改善是外因，但他内心的信心恢复是内因。数据本身不会让他重新投入——数据只是一张成绩单。他看到数据，但还没看到"这艘船知道自己往哪开"的确定性。

而且你注意到没有——你把数据甩给他，说了一句"你看，有起色了"，然后期待他眼睛一亮、原地满血复活。这不是信任重建的方式。信任重建需要持续的、透明的沟通，不是一次性的数据展示。

**行动指引**：明天别找老周聊工作。找他喝杯咖啡，聊聊他最近的状态，问他觉得产品下一步该往哪走，听听他的想法。不是要他给你答案，是要让他感觉到他的意见是被尊重的。

┃
┃ ▼ 继续追问
┃
你之前三个月有没有跟老周同步过你的思考——你发现了什么、在担心什么、打算怎么解决？还是你一直自己扛着？

---

> 阿杰第二天没找老周聊工作，而是中午拉他出去吃了碗面。老周第一次跟他讲了这半年来的真实感受：不是累，是不知道目标在哪。每次问"我们接下来要做什么"，得到的答案都是"先把留存做起来"。但没有具体目标，没有时间表，他觉得自己在一艘没有航线的船上。

```
用户：跟老周吃完那碗面之后，我把这三个月的用户调研记录和留存变化数据整理成一个文档发给了他。他回了一条消息："数据有起色，挺好。" 只有四个字，但我觉得比之前"在做"暖和多了。第二天他主动来问我，那个接口调整要什么时候上线，他可以安排在今天下午做。
```

**毛选原理**：群众路线（《关于领导方法的若干问题》L4359）。"从群众中来，到群众中去。"

**分析**：你注意到没有——从你发数据给老周到老周主动来问优先级，中间发生了什么？不是你做了什么技术调整，不是你解决了什么技术难题。你只是让他知道了你在做什么、做到了什么。信息透明本身就是管理工具。

老周回的那句"数据有起色，挺好"，不是一个冷漠的评价，是一个信号——他在重新评估这艘船是不是有船长。而他从"你问我答"变成"主动来问"，是信任重建的第一步。你不需要他立刻激情满满，你需要他愿意再看一眼、再等一等。

**行动指引**：把这件事固定成习惯——从这周开始，每周五发一封简短的产品周报给全团队，写清楚：这周发现了什么、做了什么改动、数据有什么变化、下周的计划。不需要长，五条要点就行。

┃
┃ ▼ 继续追问
┃
你现在回过头看，老周要走这件事，本质上是他的问题还是你的问题？

---

> 注册优化和新手引导上线两周后，月留存从 8% 涨到了 14%。虽然离 20% 的目标还有差距，但趋势是三个月来第一次掉头向上。阿杰给投资人发了一份简短的进展报告——这是他第一次主动给投资人同步数据，而不是等对方来问。

