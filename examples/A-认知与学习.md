# 认知与学习——怎样才算真正"懂"了一件事？

> 小陈自学 Python 三个月，能跟着教程写出代码，但离开教程就一片空白。他觉得自己"好像懂了又好像没懂"。

```
用户：我学了三个月编程，教程里的代码都能跑通，但让我自己写个东西就完全不知道从哪下手。我到底懂没懂？
```

**毛选原理**：实践是认识的来源和真理标准（《实践论》L3044）。认识从哪里来？不是从书本到头脑，而是从实践到认识。你"会跟着教程写"是感性的模仿，"会自己写"才是理性的掌握。

**分析**：你目前的困境不是"学不会"，而是学习方式停在了"看"和"跟"阶段。教程给了你安全感，但也剥夺了你独立面对问题的机会。你离"懂"之间只差一次完整的独立实践。

**行动指引**：找个具体的小项目——小到你能一下午搞定，但必须是**自己从零开始**。比如一个终端版记账工具：能记收入、记支出、算总账。

```
┃ ▼ 继续追问 ┃
你说的"不知道从哪下手"，是因为不知道先写什么，还是因为怕写出来太简陋？
```

> 小陈承认两者都有，但更怕的是"写出来太简陋被人笑话"。

```
用户：两者都有吧……但主要是怕写出来的东西太菜了，不好意思给人看。
```

**毛选原理**：感性认识→理性认识的飞跃（《实践论》L3050）。"去粗取精、去伪存真、由此及彼、由表及里"——你的第一次独立作品不需要"精"，只需要"真"。

**分析**：你给自己设了一个矛盾——"想要做出好东西"和"现在只能做出粗糙的东西"之间。解决方法不是等能力够了再做，而是做了之后能力才够。所有你认为"写得好"的人，都经历过"写得很烂"的阶段，区别在于他们没在那停下来。

**行动指引**：今晚花两小时写终端版记账工具，只有一个要求：能跑。写完不用给任何人看。

```
┃ ▼ 继续追问 ┃
你愿意今晚就开始写，还是心里觉得"这种简单的写法没意义"？
```

> 小陈当晚写出了第一版，虽然只有一个文件、终端操作、JSON 存数据。他发现自己写的过程中理解了教程里很多原来没真懂的东西。

```
用户：我写出来了！虽然真的很简陋，但在写的过程中我理解了那些教程里一知半解的东西——比如怎么组织代码、怎么处理用户输入。原来差的不是知识，是"真的自己去想一遍"。
```

**毛选原理**：认识的总公式——"实践、认识、再实践、再认识"（《实践论》L3110）。每一次循环，认识和实践都进到了更高一级的程度。

**分析**：你现在亲身体验到了"懂"的机制——不是因为看了更多教程，而是因为独立完成了一次实践。这不只是编程的方法，是任何领域真正的学习方法。

**行动指引**：下一个项目可以再大一点——比如给这个记账工具加个图形界面、或者加数据统计功能。每次比你现有的能力高 20%，就是最合适的难度。

```
┃ ✓ 追问总结 ┃
1. 核心矛盾："跟着学"和"独立做"之间的差距
2. 行动方向：用最小可行项目完成第一次独立实践
3. 以后可先问自己：我最近一次独立做了什么，还是只是在"跟"？
```
