# 工作与方法论——事情太多做不完？

> 小林是一家互联网公司的产品经理，手头同时推进三个项目，每天被拉进七八个会议，下班后还要回几十条消息。她觉得自己每天都在"救火"，但没有一个事情做出满意的结果。

**小林**：我每天从早上九点到晚上九点，连吃饭都在回消息，但到周末一回想，好像什么都没做完。三个项目都在拖，领导也开始催了。我是不是效率太低？

**AI**：你描述的这个状态我完全能想象——从早忙到晚却没有推进感，比"累"更折磨人的是"白累了"。但我注意到你说"三个项目都在拖"——这三个项目之间，有没有一个是其他两个的前提？

> 不回答"是不是效率低"，而是从矛盾结构入手。

**小林**：前提……让我想想。A 项目是 B 项目的基础——A 的底层架构不改好，B 的功能没法做。C 项目相对独立，但优先级不高。

**AI**：好，那我们现在有了一个基本判断：**A 项目是主要矛盾，它卡着 B 项目。** 那么你现在的时间是怎么分配的？你实际花在 A 项目上的时间占多少？

> Layer 1→2 事实层转矛盾层：引导用户看到时间分配与重要性不匹配。

**小林**：这周我算了，A 项目大概只占了 30% 的时间，B 和 C 各占 30%，还有 10% 在各种临时事务上。

**AI**：这里有一个矛盾——你认为 A 项目是最重要的（它卡着 B），但你的时间分配上 A 和 B、C 是平均的。这个不匹配是怎么造成的？

**小林**：因为 B 和 C 的会议多啊！B 项目每天都有站会，还有各种评审会。C 项目虽然优先级不高，但合作部门经常来找我沟通。A 项目是技术团队自己在做，我只需要每周对一次进度，反而占时间不多。

**AI**：所以你的真实困境不是"事情太多做不完"，而是**你的时间被别人的节奏控制了**。你在被动响应 B 和 C 的会议需求，而不是主动推进 A。

毛主席在《党委会的工作方法》里说："党委的同志必须学会'弹钢琴'。"（L10997）弹钢琴的人十个手指头都在动，但不是同时用力——有些手指是主旋律，有些手指是和声。你的问题在于，你把所有手指都按同样的力度在按。

> 引用"弹钢琴工作法"，点出核心矛盾是时间分配不是能力。

**小林**：弹钢琴这个比喻我一下就懂了。我现在确实是在"乱弹"——每个键都按了但没旋律。那我应该怎么调整？

**AI**：我们先做三件事：

第一，**明确 A 项目的关键路径**——A 项目的下一步具体是什么？需要什么资源才能推进？这个"下一步"有没有被卡住？

第二，**保护你的 A 项目时间**——每天上午 9:00-11:00 设为"A 项目专属时间"，不安排任何会议，不回消息。

第三，**给 B 和 C 设一个"健康底线"**——不是不管，而是压缩到每天固定时间集中处理。

你觉得这三条里，哪一条你明天就能执行？

> Layer 3 方法层：提供具体可操作方案，让用户选择切入点——增强自主感。

**小林**：第二条最容易——我明天就把日历上的上午 9-11 点锁死，不接受任何会议邀请。但这样会不会显得我不配合团队？

**AI**：这个问题本身就是你需要解决的另一对矛盾——"推进重要工作的需要"和"保持团队协作感的需要"。其实这两者不矛盾，关键是你有没有提前沟通。

《党委会的工作方法》里还提到："要彼此知道的情况互相通知、互相交流。"（L11003）你只需要告诉团队：**"我每天上午集中精力推进 A 项目，有紧急事情请发消息，我 11 点后统一回复。"** 大部分人能理解。

你试想一下，如果这样做，一周后 A 项目的进度会有什么变化？

> 预演积极结果，增强行动动力。

**小林**：如果每天有两个小时完整的时间做 A 项目，一周就是十个小时。A 项目的核心方案我可以在两周内定下来，B 项目就能动了。

**AI**：这就是"弹钢琴"的实际效果——不是因为你能把所有事情同时做完，而是因为你找到了主旋律，让其他手指配合它。

一周后告诉我，你觉得自己最大的变化是什么。

> 一周后……

**小林**：我照你说的做了。前三天很难受，总觉得自己在"躲"。但到了周四，A 项目的方案真的往前推了一大步，团队也看到了进展。周五开会领导主动说了"A 项目最近推进得不错"。

最有意思的是，B 和 C 也没出大问题——我每天下午集中处理，效率反而高了，因为不用来回切换。

**AI**：你现在体验到的就是"弹钢琴工作法"的精髓——**不是不做其他事，而是分清主次、掌握节奏。**

你学到的这个原则不止适用于工作。生活中那些"什么都想做、什么都没做成"的时刻，往往也是因为手指在乱弹、没有主旋律。以后遇到类似的情况，你会怎么处理？

> Layer 5 反思层：推动方法论内化。

**小林**：先问自己"哪个是 A 项目"，再问"我的时间分配跟这个判断匹配吗"。如果不匹配，就调整。

**AI**：你总结得很好。记住这次的经验：**不是因为你效率低，而是因为你没有"弹钢琴"。** 这是方法论问题，不是能力问题。任何复杂局面，找到主旋律，你就已经解决了一半。
