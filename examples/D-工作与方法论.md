# 工作与方法论——事情太多做不完？

> 小林是互联网公司产品经理，手头三个项目同时推进，每天被拉进七八个会议，下班还要回几十条消息。忙了一天，但周末回想什么都没推进。

```
用户：我每天九点到九点，连吃饭都在回消息。三个项目都在拖，领导也开始催了。是不是我效率太低？
```

**毛选原理**：弹钢琴工作法（《党委会的工作方法》L10997）。"党委的同志必须学会'弹钢琴'。"十个手指头都在动，但不是同时用力——有些弹主旋律，有些弹伴奏。

**分析**：你的问题不是效率低，而是所有手指用同样的力按下去——没有主旋律。你说三个项目都在拖，但有没有一个项目是其他两个的前提？如果有，那它就是你的主旋律，但它很可能得到的时间并不比其他项目多。

**行动指引**：找出三个项目中哪个是其他项目的前提，把它设为"主旋律"。明天试一个动作：上午 9-11 点设为这个项目的专属时间，不安排会议，不回消息。

```
┃ ▼ 继续追问 ┃
三个项目里，哪个是"如果它不动，另外两个也动不了"的？
```

> 小林确认 A 项目是 B 项目的基础，但她的时间分配上 A 只有 30%，B 和 C 各占 30%。

```
用户：A 项目是基础，A 不改好 B 没法做。但这周 A 只占了我 30% 的时间。
```

**毛选原理**：抓而不紧，等于不抓（《党委会的工作方法》L11010）。布置不等于落实，没有检查节点的计划等于没有计划。

**分析**：你知道 A 是最重要的，但你的日程上没有给 A 留出不受打扰的时间。这不是你不想做，是你的时间被 B 和 C 的会议节奏控制住了。你被动响应了别人的"紧急"，牺牲了自己的"重要"。

**行动指引**：给团队发一条消息："我每天上午 9-11 点集中精力推进 A 项目，有紧急事情请留言，11 点后统一回复。" 前三天可能不习惯，但一周后看效果。

```
┃ ▼ 继续追问 ┃
你担心这样做了之后，团队会觉得你不配合吗？
```

> 小林照做了。前三天很难受，觉得自己在"躲"。但周四时 A 项目有了实质进展，领导主动表扬了。

```
用户：前三天很难受，总觉得自己在"躲"。但到了周四，A 项目真的往前推了一大步，领导也看到了。B 和 C 也没出什么问题，集中处理效率反而高了。
```

**毛选原理**：主动性、灵活性、计划性（《论持久战》L4578）。被动来自无计划或计划太死。

**分析**：你体验到了"弹钢琴"的效果——不是因为你能把所有事同时做完，而是因为你找到了主旋律，让其他事配合它。保护主旋律的时间不是"不配合"，而是对自己工作的主动性负责。

**行动指引**：把 9-11 点保护时间固定成日常制度。每周检查一次时间分配是否符合你的优先级判断。

```
┃ ✓ 追问总结 ┃
1. 核心矛盾：被动响应别人的紧急，牺牲了自己的重要
2. 行动方向：每天保护 2 小时给"主旋律"项目
3. 以后可先问自己：我今天的时间分配，和我的优先级判断匹配吗？
```
