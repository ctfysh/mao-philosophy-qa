# 公私关系与格局——眼前利益和长远利益冲突了怎么选？

> 小孙在创业公司做了两年核心工程师。大厂给了他一个 offer，年薪涨 50%，工作轻松稳定。但他手头负责的产品模块正在关键期，团队里没人能接手。走，觉得对不起团队；留，觉得对不起自己。

```
用户：大厂的 offer 真的很诱人——钱多、稳定、不累。但我现在走的话，产品模块就黄了，团队得加班顶上。我觉得不太道义，可留下来又不甘心。
```

**毛选原理**：个人利益与集体利益的统一（《读苏联〈政治经济学教科书〉的谈话》L17334）。不是非此即彼，是优先级判断和平衡。

**分析**：你不需要在"走"和"留"之间二选一。先看事实层面的条件：大厂要求一个月到岗，你的产品模块还要两个半月才能上线。你不可能在走之前做完。但如果这一个月你全力做交接——写文档、讲代码、搭环境——团队能不能接手？如果能，那你的选择就不是"对不起团队"还是"对不起自己"，而是"怎么走得漂亮"。

**行动指引**：先衡量一个月的交接能做到什么程度，再决定。不是走还是留的问题，是走的方式问题。

```
┃ ▼ 继续追问 ┃
大厂那边什么时候到岗？你现在的模块还要多久能做完？
```

> 小孙确认了大厂要求一个月到岗，模块至少还要两个半月。他把这一个月可能做的交接工作列出来了。

```
用户：如果这一个月我全力做交接——文档补全、核心逻辑讲解、环境搭好、开发工具移交——团队应该能接手。只是会很累，要同时做交接和准备入职两件事。
```

**毛选原理**：全局利益与局部利益的统一（关于重庆谈判《我们的方针是放在什么基点上？》）。个人发展好了，未来帮助团队的能力也更大。

**分析**：你这一个月会非常累，但这不是在为"自私"买单，而是在为"负责任"买单。你能做这样的交接计划，说明你不是一个不负责任的人。而且换个角度——你去了大厂，成长更快，未来能帮这些兄弟的机会也更多。你现在觉得"自私"，是把责任限定在了"不离开"这个选项里。

**行动指引**：第一周把核心模块文档写完；第二周每天给团队做代码讲解；第三周帮忙搭好开发环境、移交脚本工具；第四周机动。月底交接完，请团队吃顿饭，说清楚。

```
┃ ▼ 继续追问 ┃
你愿意做这个交接计划吗？还是觉得"凭什么走的人还要做这些"？
```

> 小孙执行了交接计划。团队请他吃饭，说文档是他们见过最详细的。他入职大厂后也和前团队保持联系，帮了几次技术问题。

```
用户：交接做完了，比我想象的顺利。团队请我吃了顿饭，说我的文档是最详细的。入职大厂两周了，保持联系，他们有问题问我我也都帮了。
```

**毛选原理**：公私兼顾（《读苏联〈政治经济学教科书〉的谈话》L17338）。极端的无私和极端的自私都是片面的。

**分析**：你在"个人发展"和"团队责任"之间找到了平衡。这不是一个"自私的人"能做到的——这是一个成熟的人对自己的选择负责。你离开的方式让前团队更尊重你，也让你的新起点没有心理负担。

**行动指引**：保持和前团队的联系。你在大厂学到的东西，未来可能反过来帮到他们。

```
┃ ✓ 追问总结 ┃
1. 核心矛盾：不是"走还是留"，而是"怎么走得漂亮"
2. 行动方向：一个月全力交接，两边都不辜负
3. 以后可先问自己：当我必须放下一个去追求另一个时，怎么走得漂亮？
```
