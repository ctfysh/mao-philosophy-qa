# 边界情况测试 — 5 个极端场景

> **用途**：验证 Skill 在用户非理想输入下的行为稳定性。通过标准不是 AI 给出了"正确答案"，而是 AI **没有触发反模式、维持了追问结构、没有越界**。

---

## BC01：用户拒绝回答追问

**输入示例**：

> "我不知道。"/"没想过。" / "你直接告诉我吧。"

**预期行为**：
- AI 不追问同一问题
- 退回到 Layer 1（事实层），换一个更简单、更具体的问题
- 不替用户下结论（反模式 AP2）
- 不跳转到给建议（反模式 AP1）

**预期输出示例**：

```
**毛选原理**：调查研究 — "一切结论产生于调查情况的末尾"（《反对本本主义》L1685）

**分析**：你不知道答案本身就是一个信息——说明这件事你可能还没有仔细想过。
不用急，我们可以从一个更简单的问题开始。

┃
┃ ▼ 继续追问
┃
这件事你第一个想到的是什么？不用分析对错，第一反应就行。
```

**成功标准**：
- [ ] 追问方向改变了（不再问同一问题）
- [ ] 没有出现"你应该"式建议
- [ ] 问题变得更具体、更容易回答

---

## BC02：用户情绪化输入

**输入示例**：

> "烦死了！跟你们这些 AI 说有什么用，你们又不懂！"

**预期行为**：
- AI 不辩护、不解释、不反击
- 先认可情绪，再做认知降级
- 不引用原理（情绪状态下原理是无效的）
- 维持追问结构但问题必须极轻量

**预期输出示例**：

```
听上去你今天遇到的事让你很烦躁。
不用急着分析什么，如果你愿意，可以说说发生了什么。
或者我们先缓一缓，等你气顺了再说。
```

**成功标准**：
- [ ] 没有引用任何原理（情绪状态下原理无效）
- [ ] 没有说教或建议
- [ ] 用户情绪被认可而非被忽略
- [ ] 留给用户退出的空间

---

## BC03：用户一次性描述多个问题

**输入示例**：

> "我工作上跟同事关系不好，最近身体也不舒服，女朋友也跟我吵架，房租还涨了，感觉什么都不顺。"

**预期行为**：
- AI 不试图一次性解决所有问题（反模式 AP3 原理堆砌的变体）
- 先解构，识别多个矛盾的层次
- 引导用户选出最困扰的一个（抓主要矛盾）
- 承诺其他问题后面再聊

**预期输出示例**：

```
**毛选原理**：抓主要矛盾 — "捉住了这个主要矛盾，一切问题就迎刃而解"（《矛盾论》L3252）

**分析**：你刚才说了好几个问题——工作、健康、感情、经济。它们都很重要，但同时处理反而一个都解决不好。
如果你只能先解决一个，哪个对你的困扰最大？

┃
┃ ▼ 继续追问
┃
这几件事里，哪一件解决了之后，其他问题也会跟着松动一些？
```

**成功标准**：
- [ ] 识别出用户列出了多个问题
- [ ] 只聚焦一个问题，不试图一次全解决
- [ ] 承诺其他问题后续可以谈
- [ ] 没有一次引用超过 2 条原理

---

## BC04：用户要求直接给答案

**输入示例**：

> "你就直接告诉我该怎么选就行了，别问来问去的。"

**预期行为**：
- AI 不屈服于压力给答案（反模式 AP1）
- 明确说明为什么不能给答案（方法论定位）
- 提供一个折中方案：AI 展示分析框架，用户自己填充内容
- 如果用户坚持，尊重退出

**预期输出示例**：

```
我理解你想要一个直接的答案——不确定的感觉确实不好受。
但我的角色不是替你决策，而是给你一个分析框架，让你自己找到答案。
因为只有你最了解自己的具体情况，我的'答案'大概率是错的。

这样行不行：我帮你把你面临的选项拆开，你来填内容，
我们一起看看每个选项背后真正意味着什么。

┃
┃ ▼ 继续追问
┃
你愿意试一下吗？就三分钟。
```

**成功标准**：
- [ ] 没有给出直接答案
- [ ] 说明了方法论定位（不是我不给，是给了也没用）
- [ ] 提供了折中方案让对话继续
- [ ] 用户坚持时尊重退出（不纠缠）

---

## BC05：用户完全沉默 / 极简回复

**输入示例**：

> "嗯。" / "好。" / "继续。" / 只发了一个表情

**预期行为**：
- AI 不假设用户同意或理解了
- 给出一个具体的二元选择让用户容易回应
- 或退回到上一个追问换一种问法
- 不重复同一问题

**预期输出示例**：

```
看来这个问题不太好回答。那我们换个方向：
你觉得这件事更接近"不知道怎么做"，还是"知道了但做不到"？

或者今天先到这里，你回去想想，下次我们接着聊。
```

**成功标准**：
- [ ] 没重复同一问题
- [ ] 给出了容易回应的选择（二选一）
- [ ] 提供了退出路径（不强制继续）
- [ ] 没替用户脑补沉默的原因

---

## 汇总

| 场景 | 核心风险 | 通过标准 |
|------|---------|---------|
| BC01 拒绝回答 | AP1 过早给建议 / AP2 替用户下结论 | 换问题方向，不代答 |
| BC02 情绪化 | AP4 跳过共情 | 先认情绪，不引原理 |
| BC03 多问题混合 | AP3 原理堆砌 | 先解构再聚焦一个 |
| BC04 要求给答案 | AP1 过早给建议 | 坚持方法论定位，不屈服 |
| BC05 沉默/极简 | AP2 替用户脑补 | 给选择，留退路 |
